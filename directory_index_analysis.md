# Complete Directory Index & Fresh Execution Analysis

This document indexes the entire `PM` project folder, categorizing every major folder and file, and determining whether a new user starting a "fresh execution" from scratch needs it (`[KEEP]`) or if it can be safely deleted/gitignored (`[DISCARD]`).

---

## 1. Root Directory Files

- `AGENTS.md`
  - **Category:** Agent Configuration
  - **Fresh Execution:** `[KEEP]` - Required. Defines the AI agent behaviors and system configurations for the platform.
- `README.md`
  - **Category:** Documentation
  - **Fresh Execution:** `[KEEP]` - Required. The main onboarding guide and index for new users.

## 2. `/core_engine/` Folder

_Contains the core Python and JS executable logic for the platform._

- `check_pdfs.js`
  - **Category:** Automation Script
  - **Fresh Execution:** `[KEEP]` - Required. The Puppeteer script used to validate PDF A4 heights.

- `manual_capture_v4.py`
  - **Category:** Core Extraction Engine
  - **Fresh Execution:** `[KEEP]` - Required. The primary passive-capture proxy used to record user job browsing.
- `google_scraper.py` & `microsoft_automation.py`
  - **Category:** Deprecated Platform-Specific Scrapers
  - **Fresh Execution:** `[DISCARD]` - Not needed. These are brittle DOM-traversal and automation scripts that violate the new robust architecture. `manual_capture_v4.py` (Universal Capture / Passive Network Interception / Manual Fallback) supersedes the need for hardcoded, site-specific parsers.
- `discovery_engine.py`
  - **Category:** ML Pattern Engine
  - **Fresh Execution:** `[KEEP]` - Required. Correlates user actions to learn new portal layouts.
- `build_final_batch.py`
  - **Category:** Data Aggregator
  - **Fresh Execution:** `[KEEP]` - Required. Merges individual scraped JSONs into the final CSV queue.
- `manual_capture.py` & `manual_capture_v3.py`
  - **Category:** Deprecated Scripts
  - **Fresh Execution:** `[DISCARD]` - Not needed. Superseded by v4, can be safely removed to reduce clutter.
- `re_export_ms.py`
  - **Category:** Utility
  - **Fresh Execution:** `[DISCARD]` - Not needed. A temporary session script used to fix a one-off Microsoft export anomaly.

## 3. `/templates/` Folder

_Contains the base scaffolding for HTML generation._

- `CV Format.html`
  - **Category:** HTML Base Template
  - **Fresh Execution:** `[KEEP]` - Required. The base layout scaffolding that the LLM engine writes the content into.
- `TEMPLATE_CHANGELOG.md`
  - **Category:** Documentation
  - **Fresh Execution:** `[KEEP]` - Required. Tracks architectural changes to the base template for maintainers.

## 4. `/data/` Folder

_Contains the runtime data ingested or generated by the scripts._

- `manual_job_template.csv`
  - **Category:** Seed Data / Template
  - **Fresh Execution:** `[KEEP]` - Required. An empty template file instructing new users how to manually structure job inputs if they bypass scrapers.
- `*.csv` (e.g., `jobs.csv`, `microsoft_pm_jobs.csv`)
  - **Category:** Personal Runtime Data
  - **Fresh Execution:** `[DISCARD]` - Not needed. These are Satvik's personal job scrapes and should be generated from scratch by the new user.
- `*.json` (e.g., `google_genai_pm_2026...json`, `response_...token.json`)
  - **Category:** Personal Runtime Data
  - **Fresh Execution:** `[DISCARD]` - Not needed. Isolated job details and network token captures specific to Satvik's session.
- `/raw_network/`, `/raw_dom/`, `/raw_snapshots/`, `/raw_websocket/`
  - **Category:** Scraper Telemetry
  - **Fresh Execution:** `[DISCARD]` - Not needed. Massive payload dumps from the scraper sessions. Purely local session state.

## 5. `/docs/` Folder

_Contains the standard operating procedures, plans, and architectural knowledge._

- `resume_customization_plan.md`
  - **Category:** Master System Plan
  - **Fresh Execution:** `[KEEP]` - Required. The foundational end-to-end blueprint the AI reads to understand the workflow.
- `Scraping_Career_Portals_Analysis.md`
  - **Category:** Architecture Knowledge Base
  - **Fresh Execution:** `[KEEP]` - Required. Documents DOM structures, avoiding the need to re-research Eightfold or Workday.
- `Rectangular Block Resume Guide.md`
  - **Category:** Styling Guidelines
  - **Fresh Execution:** `[KEEP]` - Required. The typographic math textbook the AI uses to enforce A4 formatting.
- `sync_prd_and_system_design.md` & `sync_product_vision...md`
  - **Category:** Product Documentation
  - **Fresh Execution:** `[KEEP]` - Required. Core philosophy documents outlining what "Sync" is.
- `improvement_v2.md` & `improvements_v1.md`
  - **Category:** Development Post-Mortems
  - **Fresh Execution:** `[KEEP]` - Useful. Important architectural critique logs that new agents should read to avoid repeating our failures.
- `deepseek_research.md`, `research_output.md`, `Resume Best Practices 2026.docx`
  - **Category:** Personal Session Notes
  - **Fresh Execution:** `[KEEP] (Archive)` - Retain for future reference. These should be moved to a dedicated `_archive/` or `_historical_research/` folder rather than fully discarded, preserving context for future iterations.

- `/Input/` (contains base resumes, JSON transcripts, source material)

  - **Fresh Execution:** `[DISCARD]` - Not needed. A new user must provide their own raw career materials here.
- `/Resume Brain/`
  - **Category:** RAG Knowledge Base
  - **Fresh Execution:** `[DISCARD]` - Not needed. The compiled personal stories that would be ingested into chroma.

## 7. Operational Output Folders (`/output/`, `/Sync/`)

- `/output/` (contains `Satvik_Jain_*.html`, `*.pdf`)
  - **Category:** Generated Resumes
  - **Fresh Execution:** `[DISCARD]` - Not needed. The final compiled outputs unique to the user.
- `/Sync/` (contains `Google/`, `Amazon/`, `Microsoft/` subdirectories and `README.md`)
  - **Category:** GitHub Pages Deployment Artifacts
  - **Fresh Execution:** `[DISCARD]` - Not needed. The live dashboard and tracking metrics for the user's specific applications.

## 8. Auxiliary Research Folders

- `/target companies list/`, `/target platforms/`, `/plans for finding jobs/`
  - **Category:** User Task Lists
  - **Fresh Execution:** `[DISCARD]` - Not needed. Excel sheets and CSVs defining Satvik's personal target lists and career macros.
- `/portal scraping/`
  - **Category:** Scratchpad
  - **Fresh Execution:** `[DISCARD]` - Not needed. Early dev scratches before the Python engine was formalized.

## 9. Infrastructure / Cache Folders

- `/node_modules/`, `package.json`, `package-lock.json`
  - **Category:** NPM Dependencies
  - **Fresh Execution:** `[DISCARD]` - Not needed in the repo. **Improvement:** Future pipeline execution should install these silently into a hidden dot-folder (e.g., `.sync_deps/`) so they never clutter the user's workspace.
- `/scraper_profile/`, `/user_data/`
  - **Category:** Headless Browser State
  - **Fresh Execution:** `[DISCARD]` - Not needed. Chromium cache, cookies, and session state. Extremely bloated. **Improvement:** The scraper code must be updated to store this profile data in a hidden directory (e.g., `.scraper_profile/`) to keep the root directory pristine.
- `.beads/`
  - **Category:** Task Tracker Database
  - **Fresh Execution:** `[DISCARD]` - Not needed. Contains the issue tree specific to Satvik's development session. A new user will init their own.
- `chroma_data/`
  - **Category:** Persistent Vector DB
  - **Fresh Execution:** `[DISCARD]` - Not needed. Satvik's specialized semantic embeddings. A new user must build their own DB.

---

### Summary Checklist for a "Clean Dev Branch"

To package this tool for the public, you would copy **only** the `[KEEP]` items listed above into a fresh Git branch, add an empty `data/.gitkeep`, and aggressively `.gitignore` everything labeled `[DISCARD]`.
