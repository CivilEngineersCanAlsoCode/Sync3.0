**Why unsupervised vs supervised?**  
“In this project, the AI problem was mostly about generating a good starting configuration for a brand — like related keywords, handles, hashtags, and common entities — so the customer doesn’t start from a blank page. We did not treat it like a classic supervised classification problem because we didn’t have a clean labeled dataset for ‘correct keyword sets’ across every brand and industry. Also, every client can have different naming conventions and preferences. So we used a more unsupervised or weakly-supervised approach: start from knowledge sources and patterns, generate suggestions, and then let users review and edit. The goal was speed and good defaults, not a fully automated final answer.”

**How was clustering validated?**  
“For Use Case Hub, we were not doing heavy complaint clustering like my Walmart project. Here, validation was more about: do the AI-generated topics pull the right kind of data, and do the themes and dashboards make sense for common use cases. We validated quality by checking sample outputs for many brands across industries — like whether the top pulled results were relevant and whether key brand handles and hashtags were included. We also validated with internal consultants: we compared AI-generated setups with what consultants usually configure manually and checked coverage gaps and noise.”

**How did you prevent hallucination?**  
“The key idea was: AI suggestions are not final. We prevented hallucination by keeping it as ‘assistive’ and ‘editable.’ We used trusted sources for enrichment, like structured knowledge from Wikipedia, and we showed users what keywords/handles were added so they can remove anything wrong. We also avoided letting the system auto-run unlimited data pulls blindly. The system created a strong default, but with transparency and user control, because enterprise users care a lot about trust.”

**What was precision vs recall trade-off?**  
“I treated this as a precision vs recall trade-off in topic creation. If we optimize too much for recall, we include many broad keywords and we fetch a lot of irrelevant data — that creates noise and users lose trust. If we optimize too much for precision, we keep the keyword set small and we might miss important conversations. In v1, I preferred a balanced approach leaning slightly toward precision for enterprise trust — start with highly relevant keywords and official handles/hashtags, and then let customers expand if they want more coverage.”

**What was fallback if model failed?**  
“If AI suggestions were weak for a brand or industry, the fallback was the standard manual workflow — the customer or consultant can edit topics and themes directly. Also, we had pre-built industry templates for common use cases, so even if brand keyword suggestions needed edits, users still had a strong dashboard structure to start with. Basically, the product never blocks the user; AI helps you move faster, but manual control is always available.”

**How often retrained?**  
“In this product, it wasn’t like a model that needs frequent retraining on labeled data. The system depends more on knowledge enrichment and patterns, plus continuous iteration based on feedback. Practically, we improved it through periodic updates: we reviewed common failures, added missing patterns, improved entity rules, and updated templates as we learned from new client setups. So instead of ‘retrain every week,’ it was more like ‘improve the suggestion quality through releases’ based on real usage.”

**Any bias risks?**  
“Yes, there are bias risks. If the AI relies too much on public knowledge sources, smaller brands or non-English brands may get weaker suggestions. Also, some industries have different terms across regions, so the model could over-represent Western or English-centric naming. Another bias is that popular hashtags may dominate and hide niche but important keywords. To reduce this, we kept everything editable and designed templates per industry, and we expected regional clients to tune the setup. Over time, the right approach is to learn from edits customers make and improve defaults.”

TL;DR

Use Case Hub AI = assisted setup (not deep ML). No supervised training because no universal labeled set. Validated via sampling + consultant comparison. Prevented hallucination via trusted enrichment + editability. Precision vs recall balanced (lean precision). Fallback = manual + templates. Updates via iterative releases. Bias risks = language/brand size/region.
